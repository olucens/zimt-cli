# ZIMT CLI v1.0.0 — Production-Ready NestJS Scaffolding

> The secret ingredient for production-ready NestJS apps.

## What is ZIMT?

ZIMT is a CLI that eliminates the setup tax of NestJS projects.
One command gives you a fully-wired production architecture —
Prisma, Docker, JWT auth, RBAC, Redis, Swagger, tests — all connected
and ready to ship.

---

## Commands

### `zimt init` / `zimt new`
Scaffold a clean NestJS project with Prisma + PostgreSQL, Docker Compose,
global validation pipes, and a structured src/ layout.

```sh
zimt new my-app
zimt new my-app --pm bun
```

Supports **npm · yarn · pnpm · bun** — conflicting lockfiles cleaned automatically.

---

### `zimt auth`
Inject a complete JWT authentication layer into an existing zimt project.

```sh
zimt auth
# or
zimt create auth
```

Adds: JWT access + refresh tokens, RBAC guards, role decorators,
auth controller + service, user module, crypto helpers,
Prisma migrations, Swagger docs, and E2E test scaffolding.

---

### `zimt g <name>`
Generate a full resource following the Repository pattern.

```sh
zimt g posts
```

Creates: Module, Controller, Service, DTOs, Repository interface +
implementation. Injects into `app.module.ts` via **AST manipulation**
(ts-morph) — no regex, no broken imports.

---

### `zimt r <entity-or-sql>`
Add a Redis cache-aside layer to an existing service endpoint.

```sh
zimt r users
zimt r "SELECT * FROM orders WHERE status = 'active'" --ttl 60
```

Accepts an entity name or a raw SQL SELECT. Wires `@nestjs/cache-manager`
+ ioredis and updates the target service automatically.

|---|---|
| Framework | NestJS |
| ORM | Prisma |
| Database | PostgreSQL |
| Cache | Redis (ioredis) |
| Auth | JWT (access + refresh) |
| Access control | RBAC Guards + Decorators |
| API docs | Swagger / OpenAPI |
| Containers | Docker + Docker Compose |
| Testing | Jest, E2E spec scaffolding |

## Notes

- PostgreSQL is the only supported database in this release.
- Redis cache command requires an existing resource generated by `zimt g`.
